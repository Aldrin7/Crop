================================================================================
WHAT CHANGED AND WHY
================================================================================

ORIGINAL PLAN PROBLEMS:
  1. 18 hours for ~12 major subsystems — unrealistic
  2. OMML→LaTeX conversion underestimated (could eat 3+ sessions alone)
  3. 6 journals on day one — template acquisition is unpredictable
  4. Electron desktop app adds complexity with little benefit over web
  5. Visual regression testing is fragile cross-platform
  6. No buffer for things going wrong (they will)
  7. No incremental value delivery — everything works only at session 26

REVISED APPROACH:
  - Start with 2 journals, expand to 4 at most for v2.0
  - Web-first (FastAPI + React), desktop packaging deferred to v2.1
  - OMML→LaTeX replaced with simpler Pandoc-native equation handling
  - Visual regression deferred to v2.1 (manual QA sufficient for launch)
  - 5 contingency sessions built into the schedule
  - Working demo after session 12 (not session 26)

SCOPE CUTS (deferred to v2.1+):
  ✗ Electron desktop app → web-only for v2.0
  ✗ Visual regression testing → manual QA
  ✗ OMML→LaTeX full conversion → Pandoc equation passthrough
  ✗ JATS export → defer to v2.1 (PDF + DOCX + LaTeX is enough)
  ✗ PyInstaller packaging → defer to v2.1

SCOPE ADDITIONS (things the original missed + user-requested):
  ✓ 6 journal templates with real download URLs (IEEE, Springer, Wiley,
    Wiley NJD, Elsevier, MDPI, Nature) — all offline-ready after fetch
  ✓ Template download script (scripts/download_templates.py)
  ✓ Proper error messages users can understand
  ✓ Loading states and progress feedback in the UI
  ✓ Input validation (what if the DOCX is corrupt? wrong format?)
  ✓ Logging that actually helps debug problems
  ✓ A "quick export" mode for the common case (no audit, just format)

SESSION 2 — Ingestion Engine

     - Pandoc handles equations, tables, images natively in its AST
  2. Write src/ingestion/reference_parse.py

     - Parse BibTeX with bibtexparser
     - Fallback: regex for plain-text numbered refs
  3. Write tests/test_ingestion.py
     - Use fixtures/minimal.docx, with_equations.docx, with_tables.docx
     - Test metadata extraction, AST non-empty, refs parsed
  4. Generate test fixtures (scripts/generate_fixtures.py)
  5. Commit

  2. Write src/transformation/csl.py

     - Generate bibliography section
  3. Write src/transformation/crossref.py
     - Build index of figures/tables/equations
     - Replace REF fields with formatted text
  4. Add CSL files: csl/ieee.csl, csl/springer.csl

  5. Write tests/test_transformation.py
  6. Commit

  4. Write tests/test_rendering.py

WHAT GOT ADDED (THINGS ORIGINAL MISSED)
================================================================================

1. USER-FACING ERROR MESSAGES
   Original had zero consideration for what users see when things fail.
   Every error path now produces a human-readable message:
   - "This doesn't look like a Word document" (not "Invalid DOCX structure")
   - "Couldn't find any references in your document" (not "Empty bibliography")
   - "PDF generation failed — try again" (not "Tectonic exit code 1")

2. LOADING STATES + PROGRESS
   Original had no UI feedback design. Added:
   - Upload progress bar
   - Pipeline stage indicators
   - Export progress
   - Skeleton loaders

3. INPUT VALIDATION
   Original assumed valid input. Added:
   - File type checking (reject non-DOCX)
   - File size limits (prevent memory issues)
   - DOCX structure validation (is it actually a valid doc?)
   - Empty document detection

4. QUICK EXPORT MODE
   Original required full audit pipeline for every export. Added:
   - One-click "just format it" mode (skips audit, faster)
   - Full audit available as opt-in

5. PROPER LOGGING
   Original mentioned logging but had no design. Added:
   - Structured JSON logs (for machines)
   - Human-readable log viewer in UI (for users)
   - Log levels: DEBUG for devs, INFO for users, ERROR for support

SUCCESS CRITERIA (V2.0 LAUNCH)
================================================================================

A user can:
  1. Open the web UI in their browser
  2. Drag a DOCX file onto the page
  3. Select a journal (IEEE, Springer, Wiley, Elsevier, MDPI, or Nature)
  4. Click "Export"
  5. Download a properly formatted PDF, DOCX, or LaTeX file
  6. The whole process takes under 2 minutes for a typical paper
  7. If something goes wrong, they see a clear error message
  8. If the process crashes, they can resume from where it left off

That's it. That's the launch. Everything else is v2.1.
